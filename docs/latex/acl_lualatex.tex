Báo cáo này trình bày hệ thống gợi ý bài báo tiếng Việt dựa trên học biểu diễn đồ thị (Graph Representation Learning), tập trung vào xử lý \textbf{tính thưa (sparsity)} và \textbf{cold-start} trong dữ liệu tương tác kiểu bình luận/trả lời trên VnExpress. \end{abstract}

\section{Project Overview}
\label{sec:overview}

\subsection{Motivation: Why News Recommendation?}
Trong kỷ nguyên số, nền tảng tin tức đối mặt với \textbf{information overload}: số lượng bài báo mỗi ngày quá lớn khiến người dùng khó tìm nội dung phù hợp. Hệ thống gợi ý tin tức giúp:
\begin{itemize}
  \item \textbf{Cải thiện trải nghiệm:} giảm công sức lọc tin, tăng mức độ hài lòng.
  \item \textbf{Tăng retention/engagement:} nội dung đúng sở thích tăng tương tác và thời gian ở lại nền tảng.
  \item \textbf{Tác động kinh doanh:} engagement tốt hơn thường kéo theo doanh thu quảng cáo/subscription tốt hơn.
\end{itemize}

\subsection{Problem Statement}
Bài toán: với lịch sử tương tác của user (bình luận/trả lời), hệ thống gợi ý Top-$K$ bài báo liên quan mà user chưa tương tác.

\textbf{Thách thức chính} (đúng theo slides):
\begin{enumerate}
  \item \textbf{Cold-start:} user mới / bài mới không có lịch sử.
  \item \textbf{Sparse data:} đa số user đọc/tương tác rất ít.
  \item \textbf{Vietnamese NLP:} khó khăn tiền xử lý tiếng Việt, tài nguyên hạn chế.
  \item \textbf{Temporal nature:} tin có vòng đời ngắn, sở thích thay đổi theo thời gian.
\end{enumerate}

\section{Motivation: Graph Expansion Strategy}
\label{sec:graph}

\subsection{Core Idea}
Với sparsity cực cao, mô hình chỉ dựa trên tương tác user--article (G1) khó học được embedding tốt cho user có ít tương tác. Do đó, chúng tôi \textbf{enrich graph} theo 3 tầng để kiểm soát chất lượng tín hiệu:

\begin{itemize}
  \item \textbf{G1 --- Bipartite baseline:} chuẩn collaborative filtering.
  \item \textbf{G2 --- Social expansion:} thêm user--user edges dựa trên reply (homophily).
  \item \textbf{G3 --- Semantic expansion:} thêm category nodes như hub để rút ngắn path.
\end{itemize}

\subsection{Graph Definitions}
\paragraph{G1 (User--Article).}
$V = \{U, A\}$, $E=\{E_{ua}\}$. Mỗi cạnh biểu diễn một tương tác (comment/reply vào bài).

\paragraph{G2 (Add Social Reply).}
$V = \{U, A\}$, $E=\{E_{ua}, E_{uu}\}$. $E_{uu}$ được tạo từ quan hệ reply giữa hai user trong cùng thread.

\paragraph{G3 (Add Category Hub).}
$V = \{U, A, C\}$, $E=\{E_{ua}, E_{uu}, E_{ac}, E_{uc}\}$ (tuỳ cách bạn thiết kế; slide đang nhấn mạnh C như hub).

\subsection{Interpretation}
\textbf{G2} kỳ vọng tăng chất lượng lan truyền sở thích nhờ social homophily: user tương tác với nhau có thể cùng quan tâm chủ đề. \textbf{G3} tăng density nhưng có thể \emph{dilute} sở thích vì category quá rộng.

\section{Baselines (SOTA)}
\label{sec:sota}

\subsection{Models}
Chúng tôi benchmark 3 nhóm:
\begin{enumerate}
  \item \textbf{Graph GNN:} LightGCN~\cite{he2020lightgcn}.
  \item \textbf{Graph Contrastive:} SimGCL~\cite{yu2022graph}, XSimGCL~\cite{yu2023xsimgcl}, LightGCL~\cite{cai2023lightgcl}.
  \item \textbf{Content-based:} TF-IDF và PhoBERT (embedding ngữ nghĩa tiếng Việt).
\end{enumerate}

\subsection{Short Technical Summary}
\paragraph{LightGCN.}
Loại bỏ phi tuyến và weight matrix trong GCN, chỉ giữ message passing tuyến tính và average nhiều lớp:
\[
\mathbf{E}^{(k)} = \hat{\mathbf{A}} \mathbf{E}^{(k-1)}, \quad
\mathbf{E}^{*}=\sum_{k=0}^{K}\alpha_k \mathbf{E}^{(k)}.
\]

\paragraph{SimGCL / XSimGCL.}
Thay vì augment đồ thị, hai phương pháp thêm nhiễu vào embedding trong quá trình lan truyền và tối ưu InfoNCE để tạo representation robust khi sparse.

\paragraph{LightGCL.}
Bổ sung global context (ví dụ khai thác SVD/low-rank structure) để vượt hạn chế local neighbors.

\section{Proposed Method: Heterogeneous Graph Learning}
\label{sec:method}

\subsection{Motivation: Why Heterogeneous GNN?}
Các GNN dạng homogeneous xem mọi cạnh như nhau, nhưng trong dữ liệu VnExpress:
\begin{itemize}
  \item \textbf{User--Comment--Article} thể hiện \emph{explicit interest}.
  \item \textbf{User--Reply--User} thể hiện \emph{social influence/trust}.
\end{itemize}
Nếu trộn chung, mô hình có thể học sai loại tín hiệu (``Reply'' bị coi như ``Comment''). Do đó, ta cần tách từng quan hệ và fuse sau.

\subsection{HGN (Ours): Multi-Aspect Heterogeneous GNN}
Ý tưởng HGN kế thừa trực giác từ HGRec~\cite{shi2020hgrec} nhưng tối ưu theo pipeline của dự án:

\begin{itemize}
  \item \textbf{Dual-path propagation:}
  \begin{itemize}
    \item \emph{Interest path:} lan truyền trên $\mathcal{G}_{u-a}$.
    \item \emph{Social path:} lan truyền trên $\mathcal{G}_{u-u}$.
  \end{itemize}
  \item \textbf{Fusion:} cộng hoặc attention fusion (tuỳ bạn triển khai). Slide đang dùng sum.
  \item \textbf{Backbone GNN:} so sánh GAT, GraphSAGE, TransformerConv.
\end{itemize}

\paragraph{Fusion (theo slide).}
\[
\mathbf{h}_u = \mathbf{h}_{int} + \mathbf{h}_{soc}.
\]

\subsection{HCL (Ours): Heterogeneous Contrastive Learning (Failed)}
HCL cố gắng kết hợp heterogeneity với contrastive learning (BPR + InfoNCE). Trong setting thưa và cold-start, HCL không ổn định (theo kết quả slide). Báo cáo vẫn giữ HCL như một thử nghiệm để rút ra bài học về:
\begin{itemize}
  \item nhạy cảm hyper-parameter,
  \item sampling negatives,
  \item noise/temperature trong InfoNCE.
\end{itemize}

\section{Experimental Setup}
\label{sec:exp}

\subsection{Training}
Theo slide:
\begin{itemize}
  \item Epochs: 100, Batch size: 2048.
  \item Loss: BPR; CL models thêm InfoNCE.
\end{itemize}

\subsection{Evaluation Protocols}
\paragraph{Full Ranking (Cold-Start).}
Split user 80/20 (random). Với user test, rank toàn bộ bài và tính Recall/NDCG/Hit/MRR.

\paragraph{Leave-One-Out (Loo100).}
Với mỗi user test: 1 positive + 99 negatives random. Đánh giá độ nhạy ranking trong điều kiện ``nhiễu vừa phải''.

\subsection{Metrics}
\begin{itemize}
  \item \textbf{Recall@10}: độ phủ của top-10.
  \item \textbf{Hit@10}: có/không item đúng trong top-10.
  \item \textbf{NDCG@10}: thưởng cho item đúng ở vị trí cao.
  \item \textbf{MRR}: phù hợp single-target retrieval.
\end{itemize}

\section{Conclusion and Future Work}
\label{sec:conclusion}

\subsection{Conclusion}
\begin{itemize}
  \item Hoàn thiện pipeline end-to-end cho gợi ý tin tức tiếng Việt dựa trên học đồ thị.
  \item Social expansion (G2) là tín hiệu hữu ích, giúp HGN cải thiện mạnh trong Loo100.
  \item Trong Full Ranking cold-start, content-based vẫn rất mạnh; hybrid có thể là hướng triển khai thực tế.
\end{itemize}

\subsection{Limitations}
\begin{itemize}
  \item Dữ liệu dựa trên comment/reply có thể bias (nhiều người đọc nhưng không bình luận).
  \item Mô hình đồ thị tốn compute ở graph hetero dày (G2).
  \item Chưa xử lý online/dynamic update khi bài mới xuất hiện.
\end{itemize}

\subsection{Future Directions (theo slide)}
\begin{itemize}
  \item \textbf{Dynamic graph learning:} bắt kịp sở thích theo thời gian.
  \item \textbf{More meta-paths:} mở rộng node/edge (tags/author/entity) nếu có metadata.
  \item \textbf{Self-supervised learning:} thiết kế SSL ổn định hơn cho sparse regime.
  \item \textbf{Knowledge distillation:} nén model để inference nhanh.
\end{itemize}
